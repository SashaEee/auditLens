# reviews-api: ЖАЛОБЫ и отзывы клиентов (Сбер и конкуренты)

ТРИГГЕРЫ: «сколько жалоб», «на что жалуются», «темы жалоб/растут», «отзывы клиентов»,
«недовольны», «всплеск жалоб» — ВСЕГДА этот скилл.

⛔ ВАЖНО: жалоб НЕТ в основной БД auditlens (не ищи таблицы complaint/review через alsql —
их там нет, потратишь время впустую). 390 тыс. негативных отзывов banki.ru лежат в отдельной
read-only БД, доступ к ним — ТОЛЬКО через готовые API AuditLens ниже.

## Готовые ответы одним запросом
- Сколько жалоб за неделю + динамика + растущие темы (ЭТО ОТВЕТ НА ТИПОВОЙ ВОПРОС):
  curl -s "http://127.0.0.1:8000/api/reviews/anomalies?bank=%D0%A1%D0%B1%D0%B5%D1%80%D0%B1%D0%B0%D0%BD%D0%BA"
  → signals[]: label (тема), week (за 7 дн), prev_week, baseline_week (медиана прошлых
  6 недель), ratio (во сколько раз к норме), new/accel; overall: week/ratio по банку.
- Объём/доля за период: /api/reviews/overview?bank=Сбербанк&days=90
- Темы с рисками:      /api/reviews/themes?bank=Сбербанк
- Тренд по месяцам:    /api/reviews/trend?bank=Сбербанк
- Против рынка:        /api/reviews/vs-market?bank=Сбербанк
- Конкретные жалобы:   /api/reviews/feed?bank=Сбербанк&q=слово&limit=10
- Точные метки продуктов (для product=): /api/reviews/products?bank=Сбербанк

Кириллицу в URL энкодь. В ответе всегда: период, число жалоб, ×к норме, топ-темы.
Выборка: негативные отзывы banki.ru (1–2★) — упоминай это в ответе.
