# План: персональный дайджест «Обзора» (Фаза 3)

> Цель — не «ещё одна лента», а фича, которую аудитор **не захочет отключать**, потому что
> она экономит ему время и ловит важное в его зоне ответственности. «Обзор» — флагман
> инструмента, поэтому личный слой должен усилить премиальность, а не разбавить её.
> Грунтовано на реальном коде: `src/bank_audit/digest/` (writer/aggregator/news/pipeline/store),
> персонализация из [PERSONALIZATION_PLAN.md]. LLM не экономим (≈50 пользователей).

---

## 1. Продуктовый тезис: почему не отключат

Отключают персонализацию, когда она = (а) генерик («вот новости»), (б) мимо, (в) мелко
(«вы интересуетесь ипотекой» — и всё), (г) шумно. Значит делаем ровно наоборот:

| Принцип | Как реализуем |
|---|---|
| **Про мою работу, а не «вообще»** | сигнал строим из `self_description` («проверяю начисление процентов по вкладам») — самого точного описания зоны ответственности |
| **Ловит то, что я бы пропустил** | личный слой поднимает наверх события в моих темах из ВСЕХ данных (жалобы, тарифы, новости, регуляторка) |
| **Всегда с «почему вам»** | у каждого пункта — причина совпадения («ваш фокус: эквайринг ИП») |
| **Действие в один клик** | «Разобраться» / «✦ Спросить ИИ» с уже подставленным контекстом |
| **Честная тишина** | нечего сказать по моим темам → «по вашим темам сегодня тихо», а не фейк-контент |
| **Учится от меня** | «не интересно» / 👍 на карточке → корректирует профиль → человек видит, что стало точнее → не отключает |
| **Быстро и премиально** | личный слой считается заранее (кэш), рендер мгновенный, дизайн — продолжение редакторского языка |

Ключевой момент удержания — **«ага» в первую минуту**: как только человек заполнил
«чем занимаюсь», сразу показать персональный превью-блок. И **петля обучения**: реакция на
карточки уточняет профиль.

## 2. Сигнал персонализации (из чего строим «моё»)

Единый **весовой профиль тем** на пользователя (детерминированно, 0 LLM для самого веса):

```
interest_weight(topic) =
    3.0 · (topic ∈ ключевые слова self_description)      # ручное описание — максимум
  + 2.0 · (topic ∈ pinned)                                # закреплено вручную
  + 1.5 · (topic ∈ custom)                                # своя тема
  + 1.0 · counters[topic]                                 # авто из запросов (с затуханием)
  − ∞   · (topic ∈ muted)                                 # заглушено → исключить
```

`topic` — банк-слаг, продукт-слаг или ключевое слово из self_description/custom (нормализуем
через тот же словарь, что и `parse_query_signals`). Профиль пересчитывается при изменении
интересов/описания и раз в сутки. Это ядро и для ре-ранка, и для контекста LLM-сводки.

## 3. Данные: что уже есть + что добавить

### Уже есть (персонализируем существующее — это самый ценный слой)
- **Жалобы** (корпус ~390к, `rag/reviews_dash`): «новые жалобы в ВАШИХ темах/банках за неделю».
- **Тарифы** (`change_history`, `v_sber_vs_market`): «изменения в ВАШИХ продуктах».
- **Качество** (`quality_flag`): если аудитор завязан на конкретный источник.
- **Новости** (пул из 8 источников): ре-ранк под интересы.

### Добавить источники (по просьбе — чтобы охват и польза были реальными)
Критерий: релевантность аудиту + тянется напрямую через httpx (как текущие RSS/TG-web),
т.к. из контура cloud.ru живут не все. Каждый источник **тегируем измерением аудита**
(compliance / conduct / ops / fraud / product), чтобы матчить на интересы.

| Источник | Что даёт | Тег | Формат |
|---|---|---|---|
| **ФАС России** (fas.gov.ru/news) | дела/предписания против банков (реклама, навязывание) | conduct/compliance | RSS/HTML |
| **Роспотребнадзор** | защита прав потребителей фин-услуг | conduct | RSS/HTML |
| **АСВ** (asv.org.ru) | страхование вкладов, санация | product:deposit | RSS/HTML |
| **ЦБ — «Решения регулятора»** (отзыв лицензий, ограничения) | ops/compliance | по фиду ЦБ | уже частично |
| **Т-каналы по продуктам** (ипотека/карты/вклады/льготы) | продуктовая повестка под конкретные темы | product | tg-web |
| **Т-каналы по мошенничеству/схемам** | fraud-повестка (доп. к cyberpolice) | fraud | tg-web |
| **banki.ru «Народный рейтинг» / горячее** | всплески жалоб по продуктам | conduct | RSS/HTML |

> ⚠️ Перед включением каждый источник **проверить на доступность из контура** (httpx-проба);
> недоступные пометить `enabled:false`, не ронять сбор. Реестр — расширить `news.SOURCES`
> полем `dimension` (тег аудита) для матчинга.

### Инфраструктурное требование (без него ре-ранк не работает)
**Тегировать каждый news-item и сигнал** банками/продуктами/измерением аудита —
детерминированно на генерации (`detect_bank_slugs` + продуктовые ключи + dimension источника).
Кладём теги в payload секции. Это включает и персональный ре-ранк, и «почему вам».

## 4. Архитектура: общее ядро + личный слой

```
СУТКИ (МСК, 1 раз на всех):           ЛИЧНЫЙ СЛОЙ (на каждого, локальное утро):
 daily_digest (как сейчас)             personal_digest(username, local_date)
 + теги на news/сигналах        ─┐      ├─ greeting (имя+время, 0 LLM)
 + расширенный пул источников    ├───▶  ├─ personal_lead (1 insight-LLM/юзер/сутки)
                                 │      ├─ for_you[]  (детерм. ре-ранк тегов×веса)
                                 ┘      ├─ since_last[] (diff с прошлого визита)
                                        └─ quiet (если сигналов нет)
```

- **Общее ядро** — без изменений по расписанию (07:00 МСК), плюс теги и новые источники.
- **Личный слой** — считается при первом визите в локальное утро пользователя, кэш
  `personal_digest`. LLM-сводка — 1 вызов `insight_model()` на активного пользователя (≈50/сутки, ок).
- **`personal_lead`** — редакторский абзац: берёт `self_description` + топ-веса интересов +
  релевантные сигналы ядра, пишет «что важно именно вам сегодня» с числами из данных
  (не выдумывает — как headline, через реестр-кандидатов).
- **`for_you[]`** — детерминированный ре-ранк тегированных пунктов ядра по `interest_weight`,
  с полем `reason` («ваш фокус: вклады»). 0 LLM.
- **`since_last[]`** — diff по `app_user.last_seen_at`: новые жалобы/тарифы/новости в моих
  темах с прошлого захода. Даёт ощущение «система следит за моей зоной, пока меня нет».
- **Тишина** — если по темам пусто: честный блок «по вашим темам сегодня тихо · под
  наблюдением N тем», не фейк.

Триггер личного слоя — GET `/api/overview/personal` при заходе; если кэша на `local_date`
нет и `prefs.personal_digest` и время ≥ `morning_hour` → генерим (лениво, как основной digest).

## 5. UI на «Обзоре» (премиально, флагман)

Личный слой — **band сверху**, как естественное продолжение передовицы, а не отдельная вкладка.

```
┌──────────────────────────────────────────────────────────────┐
│  Доброе утро, Александр            [персонализация: вкл ⚙]     │  ← greeting + быстрый тумблер
│  ── personal_lead (Source Serif, редакторский абзац) ──        │  ← 2-4 предложения «про вас»
│                                                                │
│  В ВАШЕМ ФОКУСЕ                                     3 · для вас │  ← eyebrow-заголовок
│  ┌─ карточка ─┐ ┌─ карточка ─┐ ┌─ карточка ─┐                  │  ← for_you: причина+число+действия
│  │ ваш фокус: │ │ ваш фокус: │ │ ваш фокус: │                  │
│  │ вклады     │ │ эквайринг  │ │ ЦБ/ипотека │                  │
│  └────────────┘ └────────────┘ └────────────┘                  │
│  ↳ с прошлого визита: +2 жалобы по картам ВТБ, ЦБ-указание…    │  ← since_last (тонкая строка)
└──────────────────────────────────────────────────────────────┘
   ── дальше идёт ОБЩЕЕ ядро (передовица/пульс/риски/новости) ──   ← как сейчас, для всех
```

Принципы (сохранить премиум):
- Тот же язык: `Instrument/Source Serif` для лида, `.eyebrow` моно-метки, `--accent`,
  hairlines, `--shadow-1`, концентрические радиусы, tabular-nums.
- Личный band визуально «тёплый» (лёгкая accent-подложка/линия), но не кричит.
- Карточки `for_you` = знакомый паттерн `BfCard` (RiskGlyph, «Разобраться», «✦ Спросить ИИ»)
  + строка **reason** («ваш фокус: …») — переиспользуем существующий премиальный компонент.
- Микро-обратная связь на карточке: «не интересно» (×) → mute темы + карточка уезжает
  (subtle exit), профиль учится.
- Персонализация off → band не рендерится, «Обзор» = текущий общий (никакой пустоты).
- Первый заход после заполнения описания → band сразу с превью (эффект «ага»).
- Плейсхолдер-скелетоны, если личный слой ещё генерится (не блокируем общий).

## 6. Механики удержания (детально)

1. **Reason на каждом пункте** — никогда «просто новость», всегда «почему вам».
2. **Actionable** — «Разобраться» (drill в нужную вкладку с префильтром) и «✦ Спросить ИИ»
   (композер ИИ-аналитика с готовым промптом в контексте темы).
3. **Diff «пока вас не было»** — уникальная ценность, которой нет в общем виде.
4. **Обучение** — реакции (× не интересно / 📌 важно) правят `muted`/`pinned` → следующий
   выпуск точнее. Показать это явно: «учтём — стало точнее».
5. **Честная тишина** — не выдумываем, если по темам пусто (доверие).
6. **Экономика в UI** — как в общем digest: «личная сводка · 1 модель-вызов/сутки».
7. **Онбординг-связка** — пустой профиль → band зовёт заполнить описание (одна кнопка).

---

## 7. Экономика и надёжность

- LLM: +1 `insight_model()` на активного пользователя/сутки (≈50) + разовый ре-ген общего.
  Кэш `personal_digest(username, local_date)` — второй заход бесплатный.
- Детерминированные `for_you`/`since_last`/`greeting` — 0 LLM, мгновенно.
- Деградация: LLM-лид упал → показываем детерминированные `for_you`+`since_last` (personal_lead
  = шаблон из топ-сигнала). Личный слой никогда не роняет общий «Обзор» (Promise.allSettled,
  как уже сделано на фронте).
- Числа — только из данных/реестра-кандидатов (анти-галлюцинации, как в `headline`).

## 8. Реализация по шагам

**Бэкенд**
1. `news.SOURCES` — добавить `dimension`-тег; добавить новые источники (с httpx-пробой,
   недоступные `enabled:false`); тегировать items банками/продуктами.
2. `aggregator`/`writer` — класть теги в payload news/сигналов (для ре-ранка).
3. `userdata.interest_weight_profile(username)` — весовой профиль (§2).
4. `digest/personal.py`:
   - `build_personal(username, local_date)` → {greeting, personal_lead, for_you[], since_last[], quiet}
   - `for_you` = детерм. ре-ранк тегов×веса из последнего `daily_digest` + reason.
   - `since_last` = diff по `last_seen_at` в темах.
   - `personal_lead` = 1 insight-LLM по self_description+веса+сигналы (реестр-кандидаты).
   - кэш в `personal_digest`.
5. Эндпоинты: `GET /api/overview/personal` (ленивая генерация+кэш), `POST .../refresh`,
   `POST /api/overview/personal/feedback` (× не интересно/📌 → interests).

**Фронт (`OverviewPage`)**
6. Личный band сверху: greeting + personal_lead + «В вашем фокусе» (карточки reason) +
   since_last + быстрый тумблер. Переиспользовать `BfCard`.
7. Микро-обратная связь на карточках → feedback-эндпоинт.
8. Состояния: off / пусто (зовёт заполнить профиль) / тишина / загрузка (скелетон).

**Проверка**
9. Деплой durable + проверка в залогиненном браузере: заполнить описание → band с превью;
   ре-ранк под интересы; diff; тумблер; обратная связь меняет выдачу.

**personal_lead (LLM)** сверху. Источники — параллельно, отдельным шагом (расширяют пул, но
не блокируют личный слой).
