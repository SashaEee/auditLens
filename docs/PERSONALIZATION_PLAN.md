# План: премиальная персонализация AuditLens

> Массовое обновление: инструмент должен ощущаться «сделанным под меня», а не общим
> рабочим столом — при сохранении общего ядра (рыночные аномалии — для всех).
> Пользователей ~50 → **LLM не экономим**: можно позволить персональную генерацию каждому.
> Идентичность даёт Authentik: `X-Authentik-Username` — ключ, `X-Authentik-Name` — имя.
> Модуль «Лазейки» в этом обновлении НЕ трогаем.

---

## 1. Принцип

**Общее ядро + щедрый личный слой.** Дорогое общее (рыночные аномалии, тарифы, пул новостей)
считаем раз на всех. Поверх — **персональный слой на каждого**, и он не стесняется LLM:
персональная утренняя сводка, персональный ре-ранк, авто-профиль интересов, который система
дописывает сама. Общие аномалии остаются общими (как просил владелец), но подача, акценты и
приоритеты — под каждого.

Три источника персонализации (ключевая идея — она не только из настроек):

| Источник | Что | Как обновляется |
|---|---|---|
| **Настройки** (ручное) | имя-обращение, TZ, закреплённые/заглушённые темы, тумблеры | пользователь в профиле |
| **Авто-счётчики** (детерминированные) | веса банков/продуктов/тем из запросов | на каждый запрос, с затуханием |
| **LLM-нарратив** (авто, премиум) | живой текст «чем интересуется Александр» | LLM дистиллирует активность (после отчётов / ночью) |

Пользователь видит все три в профиле и может править — но система **сама** наполняет 2 и 3.

---

## 2. Что даёт авторизация

- **`username`** — стабильный ключ для всего: история, отчёты, интересы, настройки, шеринг.
- **`name`** — приветствие, обращение (`Александр` = первое слово), лейбл в левом нижнем углу вместо «Аудитор». Чинить кодировку (latin-1→utf-8).
- **Часовой пояс** — из браузера (`Intl.DateTimeFormat().resolvedOptions().timeZone`), с ручным override.
- **Директория пользователей** — все, у кого есть строка в `app_user` (заходили) → адресаты шеринга.

## 3. Модель данных (миграция 014, идемпотентная)

```sql
CREATE TABLE app_user (
    username      text PRIMARY KEY,             -- X-Authentik-Username
    display_name  text,                         -- декодированное имя
    timezone      text DEFAULT 'Europe/Moscow',
    prefs         jsonb NOT NULL DEFAULT '{}',  -- ручные настройки (§7)
    interests     jsonb NOT NULL DEFAULT '{}',  -- {counters, pinned, muted} (§4)
    profile_note  text,                         -- LLM-нарратив «чем интересуется» (§4)
    profile_note_at timestamptz,
    created_at    timestamptz DEFAULT now(),
    last_seen_at  timestamptz DEFAULT now()
);

CREATE TABLE chat_session (
    session_id  bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username    text NOT NULL,
    title       text,                           -- авто из первого вопроса или ручной
    pinned      boolean DEFAULT false,
    created_at  timestamptz DEFAULT now(),
    updated_at  timestamptz DEFAULT now()
);
CREATE INDEX chat_session_user ON chat_session (username, updated_at DESC);

CREATE TABLE chat_message (
    message_id  bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    session_id  bigint NOT NULL,
    role        text NOT NULL,                  -- user | assistant
    content     text NOT NULL,
    meta        jsonb DEFAULT '{}',             -- sources/charts/mode/force_deep/report_id
    created_at  timestamptz DEFAULT now()
);
CREATE INDEX chat_message_session ON chat_message (session_id, created_at);

CREATE TABLE report (
    report_id   bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username    text NOT NULL,                  -- автор
    session_id  bigint,
    question    text NOT NULL,
    title       text,
    body        text NOT NULL,                  -- финальный markdown
    payload     jsonb DEFAULT '{}',             -- charts/ranking/sources/coverage
    banks       text[],                         -- теги для «Мои отчёты» и ре-ранка
    created_at  timestamptz DEFAULT now()
);
CREATE INDEX report_user ON report (username, created_at DESC);

CREATE TABLE report_share (
    share_id     bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    report_id    bigint NOT NULL,
    owner        text NOT NULL,
    shared_with  text,                          -- username; NULL = всем пользователям инструмента
    created_at   timestamptz DEFAULT now(),
    revoked_at   timestamptz
);
CREATE INDEX report_share_to ON report_share (shared_with) WHERE revoked_at IS NULL;

CREATE TABLE user_event (
    event_id  bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username  text NOT NULL,
    ts        timestamptz DEFAULT now(),
    kind      text NOT NULL,                    -- ai_query | drill | report_open | share | tab
    payload   jsonb DEFAULT '{}'
);
CREATE INDEX user_event_user ON user_event (username, ts DESC);

CREATE TABLE personal_digest (                  -- персональная утренняя сводка, кэш
    username    text NOT NULL,
    local_date  date NOT NULL,                  -- локальная дата пользователя
    payload     jsonb NOT NULL,                 -- greeting, lead, for_you[], focus[]
    generated_at timestamptz DEFAULT now(),
    llm_model   text, tokens_in int, tokens_out int,
    PRIMARY KEY (username, local_date)
);
```

Накат — руками через psql (на проде нет `schema_migrations`, см. `docs/ARCHITECTURE.md`).

## 4. Персонализация: как профиль наполняется САМ

Три слоя `interests` + `profile_note`:

1. **Авто-счётчики** (0 LLM, на каждый запрос):
   `ai_query → user_event` с разобранными банками (`detect_bank_slugs`), продуктами (по категориям), темами (review-таксономия). Пересчёт: `new = 0.95·old + signal`, нормировка. Профиль «дышит» — фокус смещается, старое забывается.

2. **LLM-нарратив** (премиум, авто): после каждого 3-го отчёта или ночью LLM читает последние вопросы+заголовки отчётов пользователя и переписывает `profile_note` —
   *«Александр фокусируется на ипотеке Сбера и ВТБ, регулярно сравнивает ставки и первый взнос, следит за жалобами по картам»*. Это и показывается в профиле, и подаётся в персональный дайджест как контекст.

3. **Ручное** (`prefs`): пользователь может закрепить темы («всегда показывай ипотеку»), заглушить («не интересует автокредит»), поправить обращение. Ручное перекрывает авто.

Прозрачно: в профиле видны все три — счётчики (визуально), нарратив (текст, редактируемый), закреплённое/заглушённое.

## 5. Персональный дайджест (щедрый LLM, на каждого активного)

Общее ядро — 1 генерация/МСК-сутки (как сейчас). Поверх — **персональная утренняя сводка на каждого**, кэш в `personal_digest (username, local_date)`:

- **Greeting**: «Доброе утро, Александр» — по TZ и порогу «утра» пользователя.
- **Персональный lead (LLM)**: 1 вызов `insight_model()` на пользователя в день — живой абзац «что важно именно вам сегодня», построенный из `profile_note` + `interests` + общих сигналов ядра (аномалии/тарифы/новости). Пример: *«В вашем фокусе — ипотека: жалобы по Сберу выросли ×2.1 за неделю (вы это отслеживаете), а ВТБ снизил ставку по семейной на 0.4 п.п.»*. 50 юзеров = ~50 вызовов/сутки — норм.
- **«Для вас» (ре-ранк, 0 LLM)**: пул новостей ядра тегируется банками/темами на генерации; per-user скор = Σ(interest[tag]) → топ-строка с причиной.
- **Фокус-сигналы**: из общих аномалий/тарифов первыми поднимаются совпадающие с интересами («В вашем фокусе: карты ВТБ»).

Триггер — первый визит в локальное утро (или явный ⟳). Ядро при этом не пере-генерируется по поясам — экономим только на дорогом общем, персональное считаем свободно.

Тумблер `prefs.personal_digest` (дефолт on) — можно вернуться к чисто общему виду.

## 6. История чатов и отчётов — премиальный UI

Задача: дать память, не захламив чистый центрированный холст вкладки ИИ-аналитик.

**Паттерн — off-canvas левый выдвижной drawer внутри вкладки ИИ** (как в Claude/Linear), скрыт по умолчанию:

- **Триггер** — тонкая иконка-часы рядом с «Новый запрос» + горячая клавиша (`⌘K` / `⌘/`). По умолчанию скрыт → welcome-экран остаётся чистым и центрированным (премиальность сохранена).
- **Открытие** — плавный slide + лёгкий blur-backdrop (переиспользуем существующий `rail-backdrop` и `fade-in`). `Esc` закрывает.
- **Содержимое drawer — две вкладки:**
  - **«Диалоги»**: поиск сверху; группировка по времени («Сегодня», «Вчера», «На неделе», «Раньше»); каждый пункт — авто-заголовок (из первого вопроса, коротко) + время; hover открывает действия (переименовать, закрепить 📌, поделиться, удалить). Клик → сессия загружается в ленту.
  - **«Отчёты»**: премиальная галерея карточек (заголовок, дата, банки-теги, кнопки «Открыть»/«Поделиться»/PDF). Повторное открытие без пере-генерации.
- **Чистота холста**: сама лента и правый `dr-rail` источников не меняются; история живёт слева и off-canvas.
- **Микро-детали премиума**: скелетоны при загрузке, курсив-акцент на активной сессии, аккуратные разделители-времена, «Новый запрос» остаётся главной кнопкой.

Persist: хук в `/api/ai/analyze` (app.py:836) — в начале сохранить вопрос (создать/продолжить сессию), по завершении SSE аккумулировать финальный отчёт и сохранить `chat_message(assistant)` + запись `report`. Авто-заголовок сессии — из первого вопроса (можно позже LLM-переписать красиво).

Эндпоинты: `GET/DELETE /api/chat/sessions[/{id}]`, `POST /api/chat/sessions/{id}/rename|pin`, `GET /api/reports[/{id}]`.

## 7. Часовые пояса

- `app_user.timezone` (IANA) из браузера, ручной override; `prefs.morning_hour` (дефолт 7).
- Приветствие/«утро-день-вечер» и триггер персональной сводки — по TZ пользователя.
- Общее ядро — один выпуск/МСК-сутки; персональная «свежесть по утрам» — через `personal_digest (username, local_date)`, а не пере-генерацию ядра.

---

## 8. Шеринг отчётов (всем пользователям инструмента)

- Кнопка «Поделиться» на отчёте (в галерее и на открытом отчёте) → выбор адресата из **директории пользователей инструмента** (все строки `app_user` = кто заходил) ИЛИ «всем пользователям» (`shared_with = NULL`).
- Раздел «Поделились со мной» (в drawer «Отчёты» отдельной вкладкой или бейджем).
- Отзыв доступа (`revoked_at`). Автор виден получателю.
- Privacy: шеринг — всегда явное действие (клик), отзываемое, только внутри инструмента.

## 9. Фазы

| Фаза | Что | Ощущение |
|---|---|---|
| **0. Фундамент** | `app_user` + upsert-on-request + фикс имени + TZ; имя в левом углу вместо «Аудитор»; приветствие по имени+времени; `GET/PUT /api/me` | «меня узнают» — сразу |
| **1. Память** | `chat_session/message/report` + persist в `/api/ai/analyze` + premium drawer истории/отчётов (§6) | «инструмент помнит меня» |
| **2. Профиль интересов** | `user_event` + авто-счётчики + LLM-нарратив `profile_note` + просмотр/правка в профиле (§4) | «понимает, чем я занят» |
| **3. Персональный дайджест** | персональная утренняя сводка (LLM на каждого) + «Для вас» + фокус-сигналы + локальное утро (§5) | «главная — про меня» |
| **4. Панель персонализации** | настройки: TZ, обращение, закрепить/заглушить темы, тумблеры, «во сколько утро» (§7) | «я этим управляю» |
| **5. Шеринг** | кнопка «Поделиться» + директория + «Поделились со мной» + отзыв (§8) | «работаем вместе» |

Рекомендуемый порядок: **0 → 1 → 2 → 3 → 4 → 5** (профиль до персон-дайджеста, чтобы сводке было на чём строиться). Каждая фаза — отдельный durable-деплой (миграция руками + ребилд + recreate + проверка в залогиненном браузере). Спуфинг :8000 — оставляем как есть (владелец, К4).
